# 🏠 Локальная разработка и управление контентом

## Концепция

Этот проект использует **Git-based CMS** подход:
- Контент редактируется локально через админ-панель
- Изменения коммитятся в Git
- На сервере просто собирается статический сайт

## Преимущества

✅ Безопасность - нет админки на проде  
✅ Версионность - вся история изменений в Git  
✅ Простота - не нужны базы данных  
✅ Скорость - статический сайт работает быстро  
✅ Бэкапы - автоматически через Git  

## Workflow

### 1. Локальная разработка

```bash
# Запусти dev-сервер
npm run dev

# Открой админку
# http://localhost:3000/admin
# Пароль: из .env файла (ADMIN_PASSWORD)
```

### 2. Редактирование контента

1. Открой `http://localhost:3000/admin`
2. Войди с паролем
3. Редактируй страницы, контакты, создавай новые страницы
4. Сохраняй изменения
5. Проверяй результат на `http://localhost:3000`

### 3. Публикация изменений

```bash
# Закоммить изменения
git add public/content/
git commit -m "Обновил контент: добавил новую страницу"

# Запушить на GitHub
git push
```

### 4. Деплой на сервер

**Вариант А: Автоматический (рекомендуется)**

На сервере выполни:
```bash
cd /root/logos
./deploy.sh
```

**Вариант Б: Вручную**

```bash
cd /root/logos
git pull
npm run build
pm2 restart center-logos
```

## Структура файлов контента

```
public/content/
├── about.json          # Страница "О центре"
├── contacts.json       # Контактная информация
├── gallery.json        # Галерея
└── pages.json          # Список всех страниц
```

## Создание новых страниц

1. В админке нажми "Создать страницу"
2. Заполни:
   - ID: `my-page` (латиница, без пробелов)
   - Путь: `/my-page`
   - Название: `Моя страница`
3. Добавь компоненты (текст, изображения, видео и т.д.)
4. Сохрани
5. Закоммить и запушить изменения

## Бэкапы

Все изменения автоматически сохраняются в Git истории:

```bash
# Посмотреть историю изменений
git log public/content/

# Откатить изменения
git checkout HEAD~1 public/content/about.json
```

## Переменные окружения

Создай `.env` файл локально:

```env
# Telegram (для контактной формы)
TELEGRAM_BOT_TOKEN=твой-токен
TELEGRAM_CHAT_ID=твой-chat-id

# Email (Яндекс SMTP)
SMTP_USER=твой-email@yandex.ru
SMTP_PASS=твой-пароль-приложения
SCHOOL_EMAIL=твой-email@yandex.ru

# Яндекс.Карты
YANDEX_MAPS_API_KEY=твой-ключ

# Админка (локально)
ADMIN_PASSWORD=твой-пароль
```

⚠️ **Важно:** `.env` файл не коммитится в Git (он в `.gitignore`)

## Troubleshooting

### Изменения не видны локально

1. Перезапусти dev-сервер: `Ctrl+C` → `npm run dev`
2. Очисти кеш браузера: `Ctrl+Shift+R`

### Ошибка при сохранении

Проверь права на папку:
```bash
chmod -R 755 public/content/
```

### Конфликты в Git

```bash
# Если есть конфликты при git pull
git stash
git pull
git stash pop
# Разреши конфликты вручную
```

## Полезные команды

```bash
# Проверить статус Git
git status

# Посмотреть изменения
git diff public/content/

# Отменить локальные изменения
git checkout public/content/

# Создать новую ветку для экспериментов
git checkout -b new-content
```

## Автоматизация деплоя (опционально)

Можно настроить автоматический деплой через GitHub Actions:

1. При каждом push в main
2. GitHub Actions запускает деплой на сервере
3. Сайт обновляется автоматически

Подробнее в `GITHUB_ACTIONS_SETUP.md` (если нужно - создам)
